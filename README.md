Built an AI-powered E-Waste Condition Classifier from scratch using YOLOv8 + Streamlit ♻️

Every year, millions of electronic devices are dumped without proper sorting. Many reusable devices are wasted, while recyclable components never reach recycling centers efficiently. Manual inspection is slow, inconsistent, and expensive.

To address this real-world problem, I built an AI system that classifies e-waste devices into:
• Usable
• Better
• Trash

The goal is to help improve:
• E-waste segregation
• Reuse decision making
• Recycling workflows
• Sustainable disposal practices

I handled the complete ML pipeline myself:
• Dataset preparation
• Model training using YOLOv8
• Label mapping
• Inference pipeline
• Confidence scoring
• Result persistence using Streamlit session state

The app supports multiple e-waste categories including:
Battery, Mobile, PCB, Printer, Television, Keyboard, Microwave, Mouse, and more.

Technical stack:
• Python
• YOLOv8 (Ultralytics)
• Streamlit
• PIL
• Custom CSS
• JSON
• CSV logging

What I built beyond the ML model:
• Full Streamlit web application
• Dark-themed responsive UI
• Custom frontend styling
• File upload system
• Model upload integration
• Dynamic confidence visualization
• User interaction flow and state management

I also integrated location-based recycling center mapping to help users find nearby e-waste recycling facilities more easily.

The ML architecture, inference logic, mapping integration, and backend workflow were built from scratch by me.

For UI polishing, web design improvements, and frontend acceleration, I heavily used AI-assisted development tools to iterate faster.

This project taught me that solving real problems with AI requires much more than model training.
It requires combining:
ML + software engineering + UX + deployment + practical usability.
